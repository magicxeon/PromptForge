# 010-001 Character Sheet Scope and Mode Contract

**Status:** Implemented - Baseline  
**Parent:** `010-character-reference-clothing-concept.md`  
**Depends on:** Headshot MVP acceptance, saved configuration contract from 001/009

## Objective

Define the minimum useful **Character Sheet Builder** scope and its relationship to the existing `character-sheet` mode.

## Corrected Scope

The next workflow should create a reusable multi-angle character model, not a generic clothing-reference mode.

Primary output:

```text
character sheet / model sheet of the same character
front view, side view, back view
consistent identity, hair, body and outfit
```

## MVP Mode Direction

Preferred mode:

```text
character-sheet
```

The existing `character-sheet` mode should become the main implementation target.

The previously added/considered `character-reference` mode should not expand as a separate workflow unless later renamed or merged into sheet building.

## MVP In Scope

- start from saved Headshot/visual character configuration
- preserve canonical identity
- choose body/proportion visual attributes
- choose outfit preset or upload outfit front/back references
- generate front/side/back or front/back character sheet
- save generated sheet for Story Mode reuse

## MVP Out of Scope

- full scene composition
- editorial/fashion posing
- environment/background choices beyond clean sheet background
- virtual try-on guarantee
- garment layer editor
- marketplace preset selling
- multi-character sheets

## Category Policy

Character Sheet Builder should expose only:

- Character
- Face
- Hair
- Skin
- Body
- Clothing / Outfit
- Camera
- Quality

It should hide:

- Scene Story
- Environment
- Fashion Direction
- Photographic Context
- Foreground Layer
- Background Activity
- NSFW
- complex editorial pose controls

## Entry Points

Supported:

- from Headshot Builder
- from saved character configuration
- from history item generated by Headshot mode

Deferred:

- from community prompt
- from marketplace preset
- from arbitrary uploaded person image without a saved configuration

## Acceptance Criteria

- The workflow is clearly presented as Character Sheet Builder.
- It produces a reusable model sheet for Story Mode.
- It does not expose unrelated story/environment categories.
- It does not create a second identity model separate from Headshot.
