# 010-001 Scope and Mode Contract

**Status:** Draft  
**Parent:** `010-character-reference-clothing-concept.md`  
**Depends on:** Headshot MVP acceptance, saved configuration contract from 001/009

## Objective

Define the minimum useful Character Reference scope and formalize the `character-reference` mode before implementation begins.

## MVP Scope

In scope:

- start from a saved Headshot configuration or generated headshot result
- preserve canonical face identity
- choose a basic body silhouette
- choose a clothing preset or upload one clothing reference image
- choose supported clothing colors through swatches
- generate a full-body character reference or front/side/back character sheet
- save semantic selections and source lineage

Out of scope:

- exact virtual try-on
- layer/mask garment editing
- fabric simulation
- marketplace clothing assets
- multi-character sheets
- paid creator preset licensing

## Mode Name

Use a first-class mode:

```text
character-reference
```

The existing `character-sheet` mode can be reused internally only if it does not hide source ownership or prompt rules.

## Mode Responsibilities

- use canonical identity from selected saved headshot
- expose Body, Clothing Source, Clothing Preset, Clothing Color and Layout sections
- support uploaded clothing reference when provider capability allows it
- preserve admin prompt preview/edit behavior
- write generated output to normal history/comparison flows

## Entry Points

Supported MVP entry points:

- from saved headshot/profile
- from history item generated by Headshot mode
- from Studio mode switch after current headshot selections are valid

Deferred entry points:

- from community prompt
- from marketplace preset
- from uploaded character sheet without saved headshot

## Acceptance Criteria

- `character-reference` has a clear mode definition and does not overload Headshot.
- Users cannot enter the workflow without a valid canonical identity source unless they explicitly choose a blank/test mode for admin QA.
- The mode can be hidden behind a feature flag until the inventory is ready.
